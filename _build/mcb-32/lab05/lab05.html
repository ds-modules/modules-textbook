1. Introduction to Tables

Tables can be used to display datasets in a collection of columns and rows. You can think of tables in two different ways:

  • A series of columns that each contain all values for a particular attribute collected from every individual in the dataset
  • A series of rows that each contain all the data collected for an individual entry in the dataset

Creating a Table

Let's start by looking at how to create a table that contains a grocery list. We'll create three seperate lists that contain information about the item name, quantity of the item that needs to be purchased, and the price of each item.

item_name = ["Apples", "Bananas", "Carrots"]
quantities = [5, 7, 2]
prices = [0.20, 0.25, 0.75]

The lists are related by the order that entries appear. For example, the first item of our grocery list is named "Apple", has a quantity of 5, and costs $0.20. The values of this item correspond to the first entry of the item_names, quantities, and prices lists.

It can be hard to see which values go together when the data is contained in list form. To keep track of each item's features. we can create a table that uses the three lists as columns .

The following expression does the following:

  • Table() creates an empty table
  • with_columns allows you to add on columns to the empty table that was just created
  • with_columns takes on pairs of arguments. The each pair consists of the name of the new column and the list that has the contents of the new column. For example, the first column will be named "Item Name" and will contain the values in the array item_name.
  • Assigns the table to the variable groceries. From now on, we can use groceries to refer this table instead of having to constantly recreate it.
groceries = Table().with_columns(
            "Item Name", item_name, 
            "Quantity", quantities, 
            "Price", prices)

groceries

Given a table, we can get the values of a column by using 'TABLE'.column('COLUMN_NAME'). The following expression returns an array of all the items in the grocery list.

groceries.column("Item Name")

Table Operations

Much like how we can do basic artimetic in a Python interpreter, we can perform operations between the values of different columns.

Using Quantity and Price, we can find the total cost of each type of item on the grocery list by multipling the values of the columns

total_costs = groceries.column("Quantity") * groceries.column("Price")

3. Linear Regression

Often we are interested in exploring the relationship between two different things. Things are often linearly related -- any change in one variable results in a consistent, corresponding change in the other variable.

Recall that a linear relationship can be described by the equation:

$$ y = mx + b $$

where:

  • $y$ is the response variable
  • $m$ is the slope of the line
  • $x$ is the explanatory variable
  • $b$ is the intercept of the line

When we plug an $x$ value into the equation, it returns a corresponding $y$ value. In this sense, you can think of the linear equation as a conversion factor from $x$ to $y$.

When we're working with data we typically have information about the explanatory variable and response variable, but must find the slope and intercept. Linear regression can be used to estimate a linear equation that describes the relationship between the explanatory and response variables.

In least-squares linear regression, we find the line of best fit -- the straight line that minimizes the root mean square error. In layman terms, it's a "line of averages" that minimizes its distance from the observed data points.

In the graph below, the blue line is the line of best fit, and the vertical red lines represent the error of the regression line (the distance between the points and the line). Linear regression aims to minimize this error.

*** Source: UC Berkeley Data 8 Textbook https://www.inferentialthinking.com/chapters/15/ ***

The least-square regression line can be described by the equation: $$ \hat{y} = mx + b $$

where:

  • $\hat{y}$ is the estimated value of response variable
  • $m$ is the slope of the regression line. $m$ can be found using the formula $ m = r * \frac{SD_y}{SD_x} $
  • $x$ is the explanatory variable
  • $b$ is the intercept of the regression line be can be found using the formula. $b$ can be found using the formula $ b = \bar{y} - \bar{x} $

r is the correlation coefficient of the data, which describes the strength of the linear relationship.

Notice how this is similar to the general linear equation; it is also used by plugging in a value for $x$ to find the corresponding $\hat{y}$ value.

Below, we define a series of functions that can be used to easily find the features of the line of best fit.

def standard_units(xyz):
    return (xyz - np.mean(xyz))/np.std(xyz)  

def correlation(t, label_x, label_y):
    return np.mean(standard_units(t.column(label_x))*standard_units(t.column(label_y)))

def slope(t, label_x, label_y):
    r = correlation(t, label_x, label_y)
    return r*np.std(t.column(label_y))/np.std(t.column(label_x))

def intercept(t, label_x, label_y):
    return np.mean(t.column(label_y)) - slope(t, label_x, label_y)*np.mean(t.column(label_x))
